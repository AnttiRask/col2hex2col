## Project Overview

col2hex2col is an R package (on CRAN) for converting between color names and hex codes, backed by a 32,000+ color database that merges R's built-in colors with the meodai/color-names dataset.

## Common Commands

## Architecture

### Internal Data Pipeline

The package uses pre-built named vectors for O(1) lookups, stored in `R/sysdata.rda`:

- **`colornames_name_to_hex_vector`** — named vector: `color_name -> "#RRGGBB"` (used by `color_to_hex()`)
- **`colornames_hex_to_name_vector`** — named vector: `"#RRGGBB" -> color_name` (used by `hex_to_color()`)

These are generated by `data-raw/prepare_colornames.R` from `data-raw/colornames.csv` + R's `grDevices::colors()`. R colors take priority over extended colors for both directions. For hex-to-name when multiple names share a hex, the shortest extended name wins.

### Key Design Decisions

- **No hard dependencies** beyond base R. `farver` (nearest-color fallback via LAB distance), `gt` (color swatch tables), and `cli` (styled warnings) are all optional (`Suggests`).
- **R colors always win**: duplicate name resolution and hex-to-name lookups prioritize R built-in colors for backward compatibility.
- **`get_color_data()` caches** its result in a package-level environment (`.col2hex_cache_env`) to avoid recomputation.

### Source Files

- `R/color_conversion.R` — `color_to_hex()` and `hex_to_color()` (core API)
- `R/fallback_nearest_hex.R` — `fallback_nearest_hex()` (internal, LAB distance matching)
- `R/color_data.R` — `get_color_data()` (export color DB as data frame)
- `R/create_color_table.R` — `create_color_table()` (gt-based color swatch visualization)
